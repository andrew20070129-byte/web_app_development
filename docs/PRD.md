# 產品需求文件 (PRD)：線上算命系統

## 1. 專案概述
- **背景與動機**：現代人生活壓力大，常需要尋求心靈寄託或指引。透過將傳統的算命、抽籤、占卜數位化，讓使用者隨時隨地都能獲得心靈指引，並保留傳統文化中有趣的互動體驗。
- **目標用戶**：對命理主題感興趣的人、需要做決策或尋求心理安慰的群眾、喜歡在社群分享有趣測驗結果的使用者。
- **核心價值主張**：提供便捷、有趣且具備互動性的線上命理服務，將傳統宮廟或算命攤的體驗轉移到線上，並透過「線上捐香油錢」建立新型態的回饋模式。

## 2. 功能需求

### 核心功能清單
1. **算命、占卜、抽籤功能**：提供如線上擲筊、抽籤詩等互動機制。
2. **儲存算命結果**：允許使用者將覺得有意義或準確的結果保存下來。
3. **捐香油錢**：提供線上贊助或虛擬香油錢捐獻的管道，讓使用者回饋心意。
4. **使用者註冊與登入系統**：用於綁定個人的儲存紀錄與追蹤捐獻歷史。
5. **結果分享功能**：一鍵將算命或抽籤的有趣結果、籤詩分享至社群平台。

### 使用者故事 (User Stories)
- 作為 **訪客**，我希望 **能夠進行簡單的線上抽籤與占卜**，以便 **在遇到人生迷惘時獲得指引與方向**。
- 作為 **一般用戶**，我希望 **能夠儲存每次的算命與抽籤結果**，以便 **日後回顧並驗證準確度**。
- 作為 **滿意的用戶**，我希望 **能夠透過系統捐獻香油錢**，以便 **表達我的感謝並支持平台營運**。
- 作為 **一般用戶**，我希望 **能夠註冊並登入個人帳號**，以便 **跨裝置保留我的算命紀錄與捐獻明細**。
- 作為 **喜好互動的用戶**，我希望 **能將得到的籤詩分享到社群平台**，以便 **與朋友討論或展示我的幸運**。

## 3. 非功能需求
- **技術限制**：後端系統使用 Python Flask 框架，前端畫面的呈現使用 Jinja2 模板引擎進行渲染，資料庫系統採用 SQLite 進行資料儲存。
- **效能考量**：抽籤或算命的結果生成與回傳速度需小於 1 秒內，維持使用者在求神問卜時的流暢體驗。
- **安全性考量**：
  - 使用者密碼需經加密儲存（如 bcrypt）。
  - 對所有表單輸入進行防護，防止 SQL Injection 與 XSS 攻擊。
  - 對於捐香油錢相關的交易操作流程需謹慎處理，確保明細的一致性與安全。

## 4. MVP (最小可行性產品) 範圍

### Must Have (必須擁有項目)
- 核心功能的實作：線上抽籤（顯示籤詩與解析）、線上擲筊。
- 基礎系統環境：使用者註冊、登入系統。
- 個資管理：儲存個人過往算命與抽籤的歷史紀錄。

### Should Have (應該擁有項目)
- 捐香油錢功能介面（可先以簡單的點數機制或展示用的金流串接）。
- 基礎的排行榜或功德碑紀錄（顯示近期捐獻的善男信女）。

### Nice to Have (可以加分項目)
- 視覺優化：產生精美的專屬算命結果圖片以供下載。
- 第三方整合：一鍵社群分享（LINE、Facebook、Instagram）。
